# 赛博判官报告分享页文案工作台

状态：copy workbench  
用途：集中整理当前网站所有可见文案，并为后续反复打磨提供统一规范。本文关注文案质量，不描述具体代码实现。

## 使用方式

每个 UI 单元都按以下维度记录：

- **所属组件大致描述**：这个文案出现在哪里、承担什么界面功能。
- **当前文案**：页面当前实际使用的文字。
- **长度限制**：建议字数、行数或条目数量，避免视觉溢出。
- **句式风格**：短句、问句、拟态口吻、标签式等。
- **内容倾向**：应该写什么，不应该写什么。
- **实现效果**：文案在用户体验中要产生什么感受。
- **相互关系**：它和其他 UI 文案如何呼应。
- **打磨方向**：后续可以怎样优化。

## 全局文案原则

### 核心模型

左侧舞台统一遵循：

```text
场景云 = 出题
演员 = 演出
回答气泡 = 说台词
```

右侧报告统一遵循：

```text
默认态 = 赛博判官自评
名人态 = 名人点评
分享区 = 用户对外表达
```

### 语气基准

赛博判官的文案要同时具备：

- **公正感**：不吹不黑、纯路人、有一说一。
- **锋利感**：差不多得了过敏、标准不能软、先审题再锐评。
- **自洽感**：不是爱吵架，而是受不了标准缺席。
- **传播感**：能被用户截图、复述、发群聊。

### 禁忌

- 不要让名人文案像真实背书。
- 不要把赛博判官写成单纯嘴臭。
- 不要在左侧舞台塞太长解释。
- 不要让动态剧目之间出现图文错配。
- 不要让标签文案变成抽象人格术语。

## 1. 页面浏览器标题与描述

### 所属组件大致描述

浏览器标签页标题和页面 meta 描述。用户通常不直接阅读，但会影响收藏、分享预览和页面语义。

### 当前文案

```text
title：NBTI 赛博判官报告
description：你的 NBTI 工作人格是 DLAH / 赛博判官。
```

### 长度限制

- title：12-18 个中文字符以内。
- description：20-40 个中文字符以内。

### 句式风格

明确、直给、说明型。

### 内容倾向

应包含：

- NBTI
- 类型名：赛博判官
- 报告/工作人格语义

避免：

- 梗太重。
- 使用过长宣传语。

### 实现效果

让用户在浏览器标签或分享预览里知道这是一个 NBTI 类型报告页。

### 相互关系

与页面主标题 `赛博判官` 保持一致，是页面身份的最外层包装。

### 打磨方向

可选版本：

```text
NBTI 赛博判官报告
赛博判官 · NBTI 工作人格
DLAH / 赛博判官
```

### 打磨方向

## 4. 页面副标题

### 所属组件大致描述

主标题下方一句类型气质总结。

### 当前文案

```text
用逻辑拆场面，用价值下判词。
```

### 长度限制

12-18 个中文字符以内。最多一行。

### 句式风格

对仗句，动词驱动。

### 内容倾向

应总结类型的核心动作：

- 拆场面
- 下判断
- 逻辑与价值并重

避免：

- 抽象形容词堆叠，如“理性、冷静、客观”。
- 太口语化，影响标题区质感。

### 实现效果

把 `赛博判官` 从一个名字解释成一种行动方式。

### 相互关系

和右侧核心文案 `场面可以乱，逻辑不能乱` 形成呼应：

```text
标题区：用逻辑拆场面
报告区：场面可以乱，逻辑不能乱
```

### 打磨方向

可选版本：

```text
场面可以乱，逻辑不能乱。
先把题审清，再把话说狠。
把混乱拆开，把标准立住。
```

当前版本更正式，适合作为页面副标题。

## 6. 剧场拉绳按钮

### 所属组件大致描述

左侧剧场右边的拉绳控件，是触发名人参剧切换的强视觉操作。

### 当前文案

默认态：

```text

```

```text

```

### 长度限制

4 个中文字符最佳，不超过 5 个中文字符。

### 句式风格

舞台操作语，短促，有剧场感。

### 内容倾向

应表达“拉一下换幕/请演员登场”，不解释具体名人。

### 实现效果

让用户把切换行为理解为“剧场换幕”，而不是普通 tab。

### 相互关系

控制左侧舞台和右侧自评/点评同步切换。它是整页唯一主要叙事开关之一。

### 打磨方向

可选版本：

```text
名人参剧
换个演员
下一位
切换参剧
```

当前 `名人参剧 / 切换参剧` 与舞台叙事最贴合。

## 12. 默认态场景云正文

### 所属组件大致描述

默认赛博判官类型形象的出题区。

### 当前文案

```text
大家都说差不多得了。
方案看起来热闹，标准却还没站起来。
场面已经糊成一锅粥。

你怎么看？
```

### 长度限制

- 描述句：3 句。
- 问句：1 句。
- 每句 8-18 个中文字符。
- 总长度建议 45-70 个中文字符。

### 句式风格

短句叠加，最后用问句出题。

### 内容倾向

突出：

- 模糊标准
- 热闹掩盖问题
- 赛博判官被点名回答

避免：

- 太抽象，如“面对复杂问题”。
- 太长，影响舞台节奏。

### 实现效果

让默认人物登场有理由：因为场面糊了，需要赛博判官回答。

### 相互关系

必须直接引出默认态回答气泡：

```text
你怎么看？
→ “先审题。” / “再锐评。” / “标准不能软。”
```

### 打磨方向

可选版本：

```text
会议开了半小时。
大家都说方向没问题。
但没人说得清标准是什么。

你怎么看？
```

## 25A. 隐藏追加判词池

### 所属组件大致描述

隐藏操作 `追加判词` 触发后的备用短标签池。它不参与当前主舞台展示，但仍属于网站内置文案，后续如果恢复“剧场点评控制”或互动印章功能，会重新进入用户视野。

### 当前文案

```text
我先锐评
别急我在审
证据不足
建议重开
此处扣分
```

### 长度限制

- 每条 4-6 个中文字符。
- 建议保留 5-8 条，避免用户连续点击后重复太快。
- 不建议超过 8 个中文字符，否则会挤压标签视觉。

### 句式风格

标签式、短判词、带一点审判口吻。

### 内容倾向

应该像赛博判官临时盖章，而不是完整解释。可以写：

- 审题动作。
- 判断状态。
- 轻微吐槽。
- 重新评估。

避免：

- 过度攻击用户或他人。
- 太抽象的人格术语。
- 与主回答气泡重复。

### 实现效果

给页面保留一个轻量互动出口。用户点击时像在给当前场景临时加注，增强“判官正在现场审题”的感觉。

### 相互关系

它和 `不吹不黑 / 纯路人 / 有一说一` 属于同一类短标签，但功能不同：

```text
口头禅条 = 角色身份标签
追加判词池 = 临时互动反馈
回答气泡 = 正式台词输出
```

因此追加判词要更短、更像即时盖章，不能抢走回答气泡的主表达。

### 打磨方向

可以补充更有传播感的版本：

```text
先别急
题没审完
标准缺席
逻辑重连
这条重写
```

当前版本已经有角色感，但 `此处扣分` 稍偏泛用，可以后续替换成更贴近“审题庭”的说法。
