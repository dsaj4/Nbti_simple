# NBTI 报告分享页文案方向：DLAH / 赛博判官

状态：copy direction draft  
用途：为当前报告分享页提供可直接替换页面文本的文案结构。本文只定义页面该写什么，不写设计案、开发计划或实现说明。

## 文案总原则

`赛博判官` 的文案要像一场正在上演的“锐评小剧场”：舞台上发生场景，判官在下方吐出结论，卡片外再用一段有梗但准确的文字解释这个类型。

语气关键词：

- 公正口吻：不吹不黑、纯路人、有一说一、先看证据。
- 锐评口吻：拉完了、菜就多练、建议重开、这题扣分。
- 类型解释：不是为了嘴别人，而是为了把混乱现场拆成可判断的问题。
- 传播感：每段都要像能截图转发，少写抽象人格术语，多写真实场景。

避免：

- 不要把 `赛博判官` 写成纯攻击型人格。
- 不要只有“理性、客观、逻辑”这类空词。
- 不要让名人参剧变成名人背书，要写成“同类行为名场面”。
- 不要在卡片内堆长解释，长解释放在卡片外。

## 分享卡片内：舞台文本

舞台文本负责“出题”，不是解释人格。它要像一幕短剧的开场：用户一眼看到场景，就知道判官为什么要下场。

### 固定舞台标题

推荐：

```text
Cyber Court Theater
```

中文备选：

```text
赛博审题剧场
赛博锐评法庭
今日开庭：这题怎么判
```

### 普通审题剧目

四个剧目建议分别对应工作场景里的常见冲突，保留轻喜剧感。

#### 第一幕：PPT 修改案

舞台标题：

```text
第一幕：PPT 修改案
```

场景云：

```text
同事说“我只是提个小建议”，但你的 PPT 已经改了 18 版，标题、配色、目录和灵魂都不认识自己了。
```

判官吐出标签：

```text
逻辑过检
情绪扣分
建议重练
```

短判词：

```text
意见可以收，标准先写清。没有验收口径的修改，属于把大家一起拖进玄学。
```

#### 第二幕：六十秒语音案

舞台标题：

```text
第二幕：六十秒语音案
```

场景云：

```text
群里有人说“不复杂”，然后连续发了六条 60 秒语音，最后一句是“你们懂我意思吧”。
```

判官吐出标签：

```text
信息过载
证据不足
先降噪
```

短判词：

```text
懂不懂先放一边，能不能把结论、理由、动作拆开，是本庭第一个问题。
```

#### 第三幕：客观评价案

舞台标题：

```text
第三幕：客观评价案
```

场景云：

```text
朋友说“你客观评价一下就行”，你点点头，真的开始客观评价。
```

判官吐出标签：

```text
纯路人
有一说一
友谊预警
```

短判词：

```text
如果对方要的是安慰，别把评审表直接拍脸上。真话也需要包装，不然容易变事故。
```

#### 第四幕：指标缺席案

舞台标题：

```text
第四幕：指标缺席案
```

场景云：

```text
方案看起来很热闹，页面很满，词也很大，但核心指标只有一句“感觉不错”。
```

判官吐出标签：

```text
拉完了
指标缺席
菜就多练
```

短判词：

```text
没有指标的热闹，只能算氛围组。要上线可以，先把判断标准补上。
```

## 名人参剧：罗永浩幕

这一幕适合做高传播喜剧幕。重点不是嘲讽罗永浩，而是借“自打嘴巴”这个公众熟悉的名场面，解释 DLAH 的反差能力：能承认判断变化，也能继续行动。

### 舞台标题

```text
名人参剧：罗永浩的自打嘴巴
```

### 舞台场景云

```text
理想主义发布会和现实直播间同时亮起，过去的判断和现在的选择被摆到同一张桌上。罗永浩轻拍自己脸：打脸归打脸，活儿还得继续干。
```

### 判官吐出标签

```text
自打嘴巴
认真打脸
继续行动
```

### 判官回应

```text
本庭认为：前后不一致不一定是塌房，也可能是人终于被现实更新了版本。关键不在于有没有打脸，而在于打完脸以后还敢不敢把逻辑补上。
```

### 名人点评

```text
罗永浩这一幕像 DLAH 的反差面：嘴上可以很硬，标准可以很高，但一旦现实证据摆上台，也能把“以前我说”改成“现在我做”。赛博判官看重的不是永不犯错，而是犯错后还能不能公开修正、继续推进。
```

### 分享短句

```text
我的同类型名场面：罗永浩式自打嘴巴。打脸不可怕，可怕的是打完还不更新系统。
```

## 名人参剧：苏格拉底幕

这一幕适合做经典正剧幕。重点是“被审判者反向审题”：苏格拉底不是在逃避指控，而是把指控拆成更深的问题。

### 舞台标题

```text
名人参剧：苏格拉底的审判申辩
```

### 舞台场景云

```text
雅典审判现场，控诉词刚落下，苏格拉底没有急着求饶，而是把问题反问回去：什么叫败坏？什么叫虔敬？一个人到底该怎样生活？
```

### 判官吐出标签

```text
连续追问
逻辑申辩
反向审题
```

### 判官回应

```text
本庭认为：真正的审题不是回答别人给的选项，而是先判断题目本身有没有偷换概念。苏格拉底这幕不是嘴硬，是把审判现场改造成了逻辑现场。
```

### 名人点评

```text
苏格拉底这一幕像 DLAH 的底层动作：面对压力时，不先讨好场面，而是继续追问概念、证据和价值。赛博判官会被这种行为吸引，因为它把“我怎么看”推进到了“我们凭什么这么看”。
```

### 分享短句

```text
我的同类型名场面：苏格拉底式反向审题。你以为他在被审，其实他在审题。
```

## 卡片外：报告解释区核心文案

卡片外的解释区应该承担“让人看懂自己为什么是这个类型”的任务。核心是一段有梗的类型解释，后面再展开维度、优势和预警。

### 结果摘要

标题：

```text
你是会把混乱现场切开的人
```

正文：

```text
赛博判官不是单纯爱吐槽，而是看到问题时会先抓结构、再抓证据、最后给出价值判断。别人还在“感觉差不多”的雾里摸索，你已经开始问：标准是什么？证据在哪？谁在偷换概念？这件事到底该不该过？
```

### 有梗解释

```text
你的脑内像常驻了一个小型审题庭：左边坐着“不吹不黑”，右边站着“菜就多练”，中间的你负责把情绪、借口、玄学和漂亮话逐条过筛。你不是非要赢辩论，你只是受不了一件事：大家明明可以把问题说清楚，却偏要用热闹掩盖标准缺席。
```

### 优势说明

```text
你的高光时刻通常出现在复盘会、方案评审、热点讨论、产品体验吐槽和团队标准不清的时候。只要场面开始糊，你就会自动把问题拆成“事实、逻辑、价值、动作”四层，然后给出一句能让人瞬间安静的判断。
```

### 翻车预警

```text
你的风险也很明显：判得太快时，对方只会记住你的攻击感，忘了你真正想保护的是标准。锋利不是问题，问题是锋利没有递进。先让对方知道你在审事，不是在审人，你的判词才更容易被接住。
```

### 名人点评模块

标题：

```text
同类型名场面点评
```

正文：

```text
罗永浩幕对应“现实更新后的继续行动”，苏格拉底幕对应“压力现场里的反向审题”。一个偏喜剧，一个偏正剧，但它们都指向同一个 DLAH 动作：不把场面当场面，而是把场面拆成问题；不把立场当口号，而是把立场放进可检验的判断里。
```

## 右侧模块建议文案

### 四维画像

```text
D 解构：先拆问题，再谈态度。你容易发现叙事里的漏洞、偷换和空转。
L 逻辑：不太吃“感觉差不多”这一套，更相信证据链、因果和边界条件。
A 敏捷：判断速度快，场面越乱越容易进入即时裁断模式。
H 人文：判词背后不是赢辩论，而是想保住一个你认可的公平和体面。
```

### 高光场景

```text
适合你出手的地方：复盘会、方案评审、热点争议、产品体验吐槽、团队标准不清的时候。
```

### 翻车预警

```text
别把判词当结论。你的锐评很有穿透力，但太快下判会让对方只记住攻击感，忘了你真正想保护的标准。
```

```text
我的 NBTI 工作人格是「赛博判官」：不吹不黑，纯路人，有一说一。场面可以乱，逻辑不能乱；态度可以软，标准不能软。
```

备选：

```text
测出来我是 DLAH / 赛博判官。别人说“差不多得了”，我脑子里已经开庭：证据呢？标准呢？这题凭什么过？
```


## 页面文案层级

页面首屏建议只保留三层信息：

1. `赛博判官`：让用户知道结果名。
2. `用逻辑拆场面，用价值下判词。`：让用户知道类型气质。
3. `舞台场景 + 判官回答`：让用户立刻获得截图传播点。

卡片外报告区再补三层：

1. 一段有梗解释：让用户觉得“这说的是我”。
2. 名人点评：让用户觉得“这个类型有名场面可分享”。
3. 维度拆解与预警：让报告不只停留在梗。
